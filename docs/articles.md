## Layer map

The series builds in clusters. Each cluster contributes substrate to a different layer of the method:

- **Reasoning / LLM failure substrate** (#1–3) — the structural class the method is calibrated against; the case for constrained LLMs; the stack-as-intelligence frame.
- **Execution-protocol substrate** (#4–6) — the swing toward implementation; the structural retrospective; the within-session multi-mind layer.
- **Methodology substrate** (#7–11) — adversarial iteration alongside collaboration; the method as first-class subject; the recursive limit; the nudge layer; the drift audit.
- **Source-of-intent / externality substrate** (#12–13) — project inception from messy exploration; the recursive externality topology.
- **Advisor / adjudication substrate** (#14) — the repo-as-arbiter principle and chat-side failure-mode taxonomy.
- **Handoff / classification substrate** (#15) — the guardrail for expertise entering an AI-native workflow.
- **Operator-context / grounding-note substrate** (#16) — extends the grounding-note pattern to operator-side context, preserving continuity without collapsing personal memory into repo truth.
- **Relay / operative-intent substrate** (#17–#21) — source topology that names what the surfaces are for; absorption routing for classified material; operative relay as the adoption event that turns advisor output into direction; visual-inheritance as a parallel authority-preserving mechanism; recursive method self-application.
- **Generated-interface substrate** (#22) — repo + grounding note + source-of-intent files + protocols as the durable substrate from which temporary task interfaces can be generated; UX begins at the seam where intent invokes the system.
- **Consciousness / reflective-stack substrate** (#23) — philosophy-adjacent reflection on the method as a whole: consciousness as the asymptotic convergence of a constrained hall of mirrors, not a property of one layer; recursion alone is drift; the article layer is itself a reflective surface; source of intent is the gravitational center that keeps reflection oriented.

Not every article is equally repo-load-bearing. Method docs in this repo absorb structural patterns earned across the series; the article line carries reflection at its own pace. The mapping above is a navigational aid, not a hierarchy.
